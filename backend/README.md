# sttapp hosted backend

The hosted service is a Deno 2 HTTP API backed by PostgreSQL. It issues its own
desktop credentials after a Clerk browser login, derives entitlement from Dodo
webhooks, proxies only the configured Groq speech models, and records successful
usage in an append-only ledger plus an idempotent Dodo outbox. Manual-provider
mode does not call this service.

## Local development

Requirements: Deno 2.9+, Docker with Compose, and Flutter for client tests.

1. Start PostgreSQL: `docker compose -f backend/compose.yaml up -d`.
2. Copy `backend/.env.example` to an ignored local environment file and replace
   placeholders with development/test values. Never use live provider values.
3. Export the variables in your shell, then run
   `cd backend && deno task migrate`.
4. Run the API with `deno task dev` and workers with `deno task jobs`.

The default unit suite performs no network calls and needs no provider secrets:

```sh
cd backend
deno task verify
```

Database integration/migration verification:

```sh
export DATABASE_URL=postgres://sttapp:sttapp@127.0.0.1:5432/sttapp
deno task migrate
deno task test:integration
deno task migrate:down
deno task migrate
```

The committed `deno.lock` is generated by `deno task check`. Dependency changes
must update `deno.json` and the lockfile together; review registry, exact
version, integrity changes, license, and release notes. `deno task verify` also
audits the lockfile for known vulnerabilities. CI runs the same verification, a
repository secret scan, the opt-in database suite, and migration up/down/up
checks.

## Modules and contracts

- `src/config.ts`: fail-closed typed environment schema and test/live guards.
- `src/app.ts`: small dependency-injected handler and HTTP contracts.
- `src/auth.ts`: loopback/PKCE transaction, Clerk identity binding, EdDSA access
  tokens, rotating refresh tokens, reuse detection, logout, and principal
  checks.
- `src/db`: Postgres.js repository and ordered reversible migrations.
- `src/providers.ts`: mockable, timeout-bounded Dodo and Groq boundaries.
- `src/pricing.ts` / `src/flac.ts`: integer rating and bounded STREAMINFO
  parsing.
- `src/webhooks.ts`: raw-byte Standard Webhooks verification and inbox receipt.
- `src/jobs`: leased outbox, webhook processing, and retention workers.

Every JSON error uses `{error: {message,type,code,request_id}}`. Content,
credentials, request bodies, provider responses, emails, and provider keys are
excluded from application logs. `/healthz` is public liveness;
`/internal/readyz` requires the readiness bearer secret.

## Provider setup and deployment

Provider dashboard configuration is deliberately not claimed by this repository.
Before staging, create development/test Clerk and Dodo resources, subscribe both
webhook endpoints, populate the catalog IDs, and run the provider sandbox suite.
Production requires a selected host/database/job runner/secret manager, TLS,
backups with a restore drill, controlled migrations, monitoring/alerts, and the
commercial/legal gates in `Tasks.md`.

Run workers at least once per minute. Only one instance processes a worker group
at a time through `job_leases`; outbox rows recover expired leases. Deploy API
and worker code from the same revision. Run migrations before shifting traffic;
schema rollback is intended only before writes made by the new release.

## Rotation and incident basics

- Access signing key: publish old+new public keys, deploy new private `kid`,
  wait past maximum access lifetime, then remove the old public key.
- Refresh pepper: rotation invalidates sessions; schedule a reauthentication
  window rather than silently accepting two peppers.
- Clerk/Dodo webhook keys: overlap endpoints where provider support permits,
  verify delivery, then retire the old key.
- Groq/Dodo/Clerk leak: revoke at the provider, replace secret-manager value,
  redeploy, inspect redacted request IDs and usage variance, and reconcile.
- Dodo outage: durable usage remains in outbox; alert on backlog age and do not
  edit ledger history. Groq uncertain outcomes require manual reconciliation.
